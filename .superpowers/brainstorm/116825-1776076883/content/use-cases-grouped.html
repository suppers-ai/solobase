Use Cases — Grouped by Theme

All 9 original use cases plus new ones for native/binary strengths. Does this grouping make sense? Any to add, move, or cut?

🌐 Run Anywhere — Backend in the Browser (solobase-web)

The entire backend compiled to WASM. No server required.

1

Per-PR Preview Environments

PR previews as static files. Reviewers click a link, the backend runs in their browser. No containers, no databases, no teardown.

2

Offline-First Apps

Full backend in a Service Worker with OPFS persistence. Works on a plane, in the field, in emerging markets. Not "cached responses" — a real backend running locally.

3

Backend-in-the-Browser

Ship full-stack apps as static sites. Tutorials, interactive docs, product demos where prospects actually use the product.

4

Zero-Infrastructure E2E Testing

Playwright/Cypress against solobase-web in headless Chrome. Each test gets a fresh isolated backend. No docker-compose, no port conflicts.

5

Privacy-First / Data Sovereignty

User data never leaves the browser. No third-party cloud, no data residency concerns. GDPR/HIPAA-friendly by architecture, not policy.

⚡ Single Binary — Deploy and Forget (solobase native)

One Rust binary. Everything included. Incredibly cheap to run.

6

Launch a SaaS in One Binary

Auth, payments, admin panel, user dashboard, file storage, database. Deploy to a $5/mo VPS. No microservices, no managed databases, no DevOps team.

7

Ridiculously Cheap to Run

NEW. A single Rust binary uses ~20MB RAM idle. Run a full production backend on a $5 VPS that would cost $50-200/mo on a typical stack (managed DB + app server + cache + CDN). Rust's zero-cost abstractions mean you're not paying for a runtime, a GC, or an ORM. Scale vertically before you ever need to think about horizontal.

8

Local-First Development

Download binary, run it. Full backend on localhost in seconds. No Docker, no database install, no cloud account. Develop on a plane. Same code in prod.

9

AI Agent Infrastructure

An agent spins up solobase (single binary, zero config), gets auth + DB + storage + API instantly. The agent IS the developer. A2A messaging for multi-agent coordination.

🔒 Platform — Edge, Isolation & Trust (solobase on Cloudflare)

Run at the edge. Sandbox third-party code. Scale to millions.

10

Multi-Tenant Platform (Workers for Platforms)

Each customer gets their own isolated solobase instance on Cloudflare. Automatic subdomain provisioning, per-tenant config, deployment lifecycle. No Kubernetes, no shared-database compromises.

11

Safe Third-Party Extensions (WASM Sandboxing)

NEW. Let third parties build extensions/blocks that run inside your platform — but sandboxed in WASM. They can't access your env vars, can't make unauthorized network calls, can't touch other tenants' data. Everything is isolated by default. Think app marketplace (like Shopify apps) but where malicious code literally cannot escape the sandbox. Services like OpenClaw could run third-party legal automation without risk.

12

Edge Computing — Global, Sub-50ms

NEW. Deploy solobase to Cloudflare's 300+ edge locations. Your backend runs within 50ms of every user on Earth. Not just a CDN in front of a central server — the actual backend logic executes at the edge. D1 for database, R2 for storage, all colocated. Perfect for latency-sensitive apps: real-time collaboration, gaming backends, financial tools.
